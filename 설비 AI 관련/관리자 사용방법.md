## 4. 지원 데이터 파일 형식

### CSV / TXT 형식 (권장)

| 항목 | 규격 |
|------|------|
| 구분자 | 세미콜론 (`;`) |
| 인코딩 | cp949 (EUC-KR) |
| 헤더 | **2행** 구조 |
| 1행 | 인덱스 ID (`[115:0]`, `[115:1]` ...) |
| 2행 | 설비 설명 (`#01_A_소성로 RHK_SCR H1U 전류(A) (D7000)`) |
| 시간 형식 | `DD.MM.YYYY HH:MM:SS.ffffff` (예: `14.09.2025 00:16:15.030000`) |
| 소수점 구분자 | 점 (`.`) |

### Excel 형식 (.xlsx)

- 1행 헤더: 설명 컬럼명만 있을 경우 자동 인식
- 2행 헤더: 1행이 `[115:0]` 형태로 시작하면 CSV와 동일 구조로 처리

## 5. 신규 설비 추가 방법

`preprocess.py` 상단의 `EQUIPMENT_PATTERNS` 리스트에 튜플 추가:

```python
EQUIPMENT_PATTERNS = [
    ('RHK-A',    r'^#01_A_'),
    ('RHK-B',    r'^#01_B_'),
    ('PTK-히터', r'^#01_.+PTK'),
    ('NCF1',     r'^\[NCF1\]_|^NCF1\s'),
    ('PNCF1',    r'^PNCF1\s|^PNCF01\s'),
    ('RHK#A',    r'^RHK#A\s'),
    ('RHK#B',    r'^RHK#B\s'),
    # ↓ 신규 설비 추가 예시
    ('분쇄기',   r'^#03_분쇄기_|^#03_NCM_'),
]
```

- 첫 번째 원소: 대시보드에 표시될 설비명
- 두 번째 원소: Python 정규식 (컬럼명 앞부분과 매칭)

## 7. 트러블슈팅

### 인코딩 오류 (UnicodeDecodeError)

- 원인: 파일이 cp949가 아닌 다른 인코딩일 경우
- 해결: `preprocess.py`의 `_load_csv()` 함수에서 `encoding='cp949'`를 `encoding='utf-8-sig'`로 변경

### 메모리 오류 (MemoryError)

- 원인: 컬럼 수가 너무 많거나 행 수가 많은 파일
- 해결: `clean_numeric()`에서 float32 변환 추가:
  ```python
  for col in df.columns:
      df[col] = pd.to_numeric(df[col], errors='coerce').astype('float32')
  ```

### 설비가 드롭다운에 나타나지 않음

- 원인: 해당 설비의 모든 센서 값이 0 (미가동 상태)
- 해결: 정상 — `extract_equipment()` 함수는 0값 설비를 의도적으로 제외

### 컬럼명 인식 실패 (설비가 '기타'로 분류됨)

- 원인: `EQUIPMENT_PATTERNS`에 해당 컬럼명 패턴이 없음
- 해결: 컬럼명 예시를 확인하고 `EQUIPMENT_PATTERNS`에 패턴 추가

### Streamlit 캐시 문제 (데이터가 업데이트되지 않음)

```bash
# 캐시 초기화 후 재실행
streamlit cache clear
streamlit run dashboard.py
```

## 8. 성능 최적화

대용량 파일(5,000 컬럼 이상)에서 속도가 느릴 경우:

1. **float32 사용**: `clean_numeric()` 내 float64 → float32 변환
2. **데이터 리샘플링**: 1분 → 5분 간격으로 다운샘플링
   ```python
   df = df.resample('5T').mean()
   ```
3. **이상 탐지 제한**: 분산이 낮은 컬럼은 자동 스킵됨 (기존 동작 유지)
